# 架构说明 (Architecture)

## 1. 系统架构

### 核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                      Edge Node                               │
│  ┌─────────────────┐    ┌─────────────────┐                 │
│  │  OpenClaw       │    │  memory-lancedb │                 │
│  │  Gateway        │───▶│  -pro Plugin    │                 │
│  └─────────────────┘    └────────┬────────┘                 │
│           │                       │                         │
│           │                       ▼                         │
│           │              ┌─────────────────┐                │
│           │              │  LanceDB        │                │
│           │              │  (Local Store)  │                │
│           │              └─────────────────┘                │
└───────────┼─────────────────────────────────────────────────┘
            │
            │ HTTP API (Ollama)
            ▼
┌─────────────────────────────────────────────────────────────┐
│                  Inference Provider                          │
│  ┌─────────────────┐    ┌─────────────────┐                 │
│  │  Ollama Server  │───▶│  qwen3.5:2b     │                 │
│  │  (0.0.0.0:11434)│    │  (LLM Model)    │                 │
│  └─────────────────┘    └─────────────────┘                 │
└─────────────────────────────────────────────────────────────┘
```

### 数据流

1. **消息采集**: OpenClaw Gateway 接收用户消息，触发 `agent_end` 钩子
2. **异步处理**: 插件立即返回，不阻塞用户响应
3. **噪音过滤**: `noise-filter.ts` 过滤包含 "测试/test" 等关键词的消息
4. **LLM 分类**: 调用远程 Ollama 进行 worth/category/importance 分类
5. **向量化**: 使用文本嵌入模型生成向量
6. **去重检查**: LanceDB 检索相似度，避免重复存储
7. **持久化**: 符合条件的数据写入本地 LanceDB

## 2. 关键设计决策

### A. 异步非阻塞 (Async Fire-and-Forget)

**问题**: LLM 分类耗时 500ms-2s，如果同步处理会严重影响用户体验。

**解决方案**:
```typescript
onAgentEnd(event: AgentEndEvent) {
  // Fire and forget - 立即返回，不等待
  this.asyncCaptureAndStore(event).catch(console.error);
  return { status: "success" };
}
```

**优势**:
- 用户获得即时响应
- 后台静默完成复杂计算
- 单点故障不影响主流程

### B. 推理负载卸载 (Inference Offloading)

**场景**: 边缘设备（如 2 核 VPS）无法流畅运行 2B+ 参数模型。

**架构**:
- 边缘节点: 仅运行 Gateway 和轻量级插件
- 推理节点: 4 核+ CPU，运行 Ollama 服务

**配置示例**:
```json
{
  "llmClassification": {
    "provider": "ollama",
    "baseURL": "http://your-inference-server:11434",
    "model": "qwen3.5:2b"
  }
}
```

### C. 智能分类与过滤

**分类维度**:
- `worth`: boolean - 是否值得存储
- `category`: fact | decision | entity | preference | other
- `importance`: 0.0-1.0 评分
- `reason`: 分类理由

**噪音过滤**:
- 自动过滤 "测试"、"test" 等关键词
- 支持自定义过滤规则

**语义去重**:
- 使用向量相似度检测
- 阈值设为 1.0（完全匹配才跳过）

## 3. 性能指标

| 指标 | 数值 | 说明 |
|------|------|------|
| 分类延迟 | ~800ms | qwen3.5:2b 平均响应时间 |
| 用户感知延迟 | 0ms | 异步处理，无阻塞 |
| 存储去重率 | ~30% | 重复对话自动跳过 |
| 内存占用 | ~150MB | LanceDB + 插件运行时 |

## 4. 故障处理

### 网络中断
- Ollama 不可用时，自动降级为跳过分类
- 本地存储不受影响

### 模型不可用
- 返回默认分类结果 `{ worth: false, ... }`
- 记录错误日志，不中断流程

### 存储故障
- LanceDB 初始化失败时抛出错误
- 建议在初始化时检查磁盘空间

## 5. 安全考虑

### 数据传输
- 内网 HTTP 通信（可配置 HTTPS）
- 建议通过 VPN 或私有网络连接

### 数据存储
- 向量数据本地存储
- 支持加密存储（需 LanceDB 企业版）

### 敏感信息
- 支持配置敏感词过滤
- 自动检测并跳过密码等敏感内容
